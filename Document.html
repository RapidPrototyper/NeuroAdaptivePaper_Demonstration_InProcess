1. System Overview

This is a web-based Brain-Computer Interface (BCI) experiment that simulates neuroadaptive cursor control based on the research paper "Neuroadaptive technology enables implicit cursor control based on medial prefrontal cortex activity" (Zander et al., 2016).

🎯 Purpose: The system demonstrates how a cursor (robot) can learn to move toward a target based on user's brain activity patterns, with movements classified and recorded for offline EEG analysis.

Key Features

  • 3D Visualization: Three.js-powered 3D grid environment with animated robot and target
  • Three Experiment Phases: Calibration (random), BCI (implicit learning), Manual (explicit feedback)
  • Machine Learning Model: Direction probability weights updated based on user feedback
  • EEG Synchronization: All movements and events sent to LSL stream for offline analysis
  • Dual Movement Classification: Each movement classified by direction (toward/away/sideways) and quality (very good/neutral/very bad)
  • Real-time Visualization: Bar chart showing current direction probabilities
  • Gray Square Status Indicator: Physical indicator showing experiment state

4. HTML Layer (index.html)

The HTML file defines the complete user interface structure, including the experiment configuration screen and all HUD panels that appear during the experiment.

4.1 Introduction Screen

The starting screen where users configure the experiment before beginning:

  • Experiment Information: Description of the three phases and their purpose
  • Controls Summary: Key bindings (V/B for manual phase, H for HUD, Space for breaks)
  • Configuration Options: Grid size (4x4, 6x6, 8x8), experiment condition, calibration jumps, BCI targets

All panels are hidden by default and appear when pressing the 'H' key:

Statistics Panel

  • Current phase (Calibration/BCI/Manual)

  • Current move count in trial
  • Robot position (x, y)
  • Target position (x, y)

Model Visualization Panel

  • Bar Chart: Canvas-based visualization showing probability for each of the 8 directions
  • Numerical Grid: Direction probabilities displayed as percentages

Controls Panel

  • Shows "ACTIVE" during manual phase, "INACTIVE" during other phases
  • Displays key bindings (V = Acceptable, B = Not acceptable)

  • Textarea showing all generated event markers with timestamps

7. Python Bridge (lsl_bridge.py)

The Python bridge acts as a WebSocket server that receives markers from the browser and forwards them to the LSL network.

7.1 LSL Stream Setup

def setup_lsl_stream(self):
    info = StreamInfo(
        name='NeuroCursor_Markers',      # Stream name in LabRecorder
        type='Markers',                   # Stream type
        channel_count=1,                  # Single channel
        nominal_srate=0,                  # Irregular sampling
        channel_format='string',
        source_id='neurocursor_experiment'
    )
    self.outlet = StreamOutlet(info)

7.2 Message Handling

async def handle_client(self, websocket, path):
    async for message in websocket:
        data = json.loads(message)
        
        label = data.get('label')
        cls1 = data.get('cls1')
        cls2 = data.get('cls2')
        classify_now = data.get('classifyNow')
        button = data.get('button')
        
        if label:
            self.outlet.push_sample([label])              # No prefix
        if cls1:
            self.outlet.push_sample([f"cls1:{cls1}"])     # With prefix
        if cls2:
            self.outlet.push_sample([f"cls2:{cls2}"])
        if classify_now:
            self.outlet.push_sample([f"classifyNow:{classify_now}"])
        if button:
            self.outlet.push_sample([button])             # Button code only
        
        await websocket.send(json.dumps({"status": "received"}))

8. Movement Classification Algorithm

This is the core algorithm that determines how each movement is categorized for the LSL stream.

8.1 Angle Calculation

function calculateAngleToGoal(fromPos, toPos) {
    const jumpDir = {
        x: toPos.x - fromPos.x,
        y: toPos.y - fromPos.y
    };
    
    const goalDir = {
        x: targetPos.x - fromPos.x,
        y: targetPos.y - fromPos.y
    };
    
    const dotProduct = jumpDir.x * goalDir.x + jumpDir.y * goalDir.y;
    const jumpMagnitude = Math.sqrt(jumpDir.x * jumpDir.x + jumpDir.y * jumpDir.y);
    const goalMagnitude = Math.sqrt(goalDir.x * goalDir.x + goalDir.y * goalDir.y);
    
    if (jumpMagnitude === 0 || goalMagnitude === 0) return 0;
    
    const cosAngle = dotProduct / (jumpMagnitude * goalMagnitude);
    const clampedCos = Math.max(-1, Math.min(1, cosAngle));
    const angleDeg = Math.acos(clampedCos) * (180 / Math.PI);
    
    return Math.round(angleDeg);
}

8.2 Dual Classification

function classifyAngle(angle) {
    // CLASSIFICATION 1: toward/away/sideways
    let cls1;
    if (angle < 45) {
        cls1 = 'toward';      // Movement heading toward target
    } else if (angle > 100) {
        cls1 = 'away';        // Movement heading away from target
    } else {
        cls1 = 'sideways';    // Movement perpendicular to target
    }
    
    // CLASSIFICATION 2: very good/neutral/very bad
    let cls2;
    if (angle < 1) {
        cls2 = 'very good';   // Perfect alignment (0°)
    } else if (angle > 135) {
        cls2 = 'very bad';    // Almost opposite direction
    } else {
        cls2 = 'neutral';     // Moderate deviation
    }
    
    return { cls1, cls2 };
}
📊 Why Two Classifications?
cls1 (Direction): Represents whether the movement was toward, away from, or sideways to the target. Used to train BCI to recognize directional intention.
cls2 (Quality): Represents how optimal the movement was. Very good = perfect alignment, very bad = moving opposite direction, neutral = acceptable but not optimal.

90° sideways neutral Perpendicular movement

8.4 Marker Format

function createJumpMarker(fromPos, toPos, direction, classification) {
    // Format: 4x4;g41;j084:33>34;ang001;cls1:away;cls2:very bad;phase:calibration
    const marker = `${gridSize}x${gridSize};g${targetPos.x}${targetPos.y};j${String(jumpCounter).padStart(3, '0')}:${fromPos.x}${fromPos.y}>${toPos.x}${toPos.y};ang${String(angle).padStart(3, '0')};cls1:${classification.cls1};cls2:${classification.cls2};phase:${phase}`;
    return marker;
}

Marker Components:

  • 4x4 - Grid dimensions
  • g41 - Target position (row 4, column 1)
  • j084:33>34 - Jump number 84, moving from (3,3) to (3,4)
  • ang001 - Angle of movement relative to target (0-180°)
  • cls1:away - Direction classification
  • cls2:very bad - Quality classification
  • phase:calibration - Current experiment phase

9.1 Phase Details

Calibration Phase

  • Robot moves in random directions (no learning)
  • No user input required
  • Each movement is classified and sent to LSL
  • Establishes baseline movement patterns
  • Gray square shows medium gray

BCI Phase

  • Robot moves using weighted probabilities from user model
  • Initial model has equal probabilities for all directions (12.5% each)
  • No explicit user feedback
  • Additional "classifyNow" markers sent for BCI processing
  • Gray square shows light gray

Manual Phase

  • Robot moves using weighted probabilities from user model
  • User provides explicit feedback: V (acceptable) or B (not acceptable)
  • Model updates with learning rate of 0.25 per feedback
  • Gray square shows dark gray

9.2 Break Schedule

During BCI and Manual phases, break screens appear every 5 completed targets:

  • Shows progress bar with completed targets
  • Requires Spacebar press to continue
  • Gray square turns black during break
  • Break counter resets after each break

11. Data Flow Summary

  1. User clicks "Start Experiment"
  2. Experiment initializes Three.js, loads robot model
  3. Phase starts (Calibration/BCI/Manual)
  4. Robot moves to new position
  5. Movement is classified (classifyAngle() → cls1, cls2)
  6. Markers sent to LSL bridge via WebSocket:
    • Full marker string (label)
    • Direction classification (cls1)
    • Quality classification (cls2)
    • classifyNow (BCI phase only)
    • Button presses (V=50001, B=50002)
  7. LSL bridge forwards to LSL stream "NeuroCursor_Markers"
  8. LabRecorder records markers synchronized with EEG
  9. Phase complete → transition to next phase
  10. All phases complete → show completion screen
